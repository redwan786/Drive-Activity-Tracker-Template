# Drive / Folder Activity Tracker — Setup Guide

This tool tracks a whole drive or folder and logs every file/folder
**add · delete · rename · move** to a clean GitHub repo, with an
auto-generated `README.md` index and a premium `CHANGES.md` change-log.

---

## Layout (how it looks once set up)

```
<your drive or folder>\          <- this is the tracked root (.git lives here)
│   README.md                    <- auto-generated index (GitHub landing page)
│   .gitignore                   <- auto-created; keeps everything except the tracker out of git
│
└── tracker\                     <- all the machinery lives here
        #1- INITIAL.cmd          <- double-click: first-time setup
        #2- UPDATE_CHANGES.cmd   <- double-click: refresh README + CHANGES, push
        #3- RESET_CHANGES.cmd    <- double-click: wipe the change-log, fresh start
        INITIAL.ps1
        UPDATE_CHANGES.ps1
        RESET_CHANGES.ps1
        SETUP.md                 <- this guide
        CHANGES.md               <- the change-log
```

> `README.md` and `.gitignore` **must** sit at the root (git + GitHub require it).
> You never edit them by hand — both are generated/maintained automatically.
> If `.gitignore` ever gets deleted, the next run recreates it.

## Which files are required?

| File (inside `tracker\`) | Required? | Notes |
|---|---|---|
| `INITIAL.ps1` | ✅ **Required** | first-time setup logic |
| `UPDATE_CHANGES.ps1` | ✅ **Required** | the everyday update/push engine |
| `RESET_CHANGES.ps1` | ⚪ Optional | only needed if you want the "wipe change-log" feature |
| `#1- INITIAL.cmd` / `#2- UPDATE_CHANGES.cmd` / `#3- RESET_CHANGES.cmd` | ⚪ Optional | just double-click wrappers around the `.ps1`; can be regenerated anytime |
| `SETUP.md` | ⚪ Optional | this guide |
| `CHANGES.md` | ⚙️ Auto | created on first run |

| File (at the **root**) | Required? | Notes |
|---|---|---|
| `.gitignore` | ⚙️ Auto | **auto-created** by the scripts if missing — never download it |
| `README.md` | ⚙️ Auto | generated by `UPDATE_CHANGES.ps1` |

**Bare minimum to run:** just `INITIAL.ps1` + `UPDATE_CHANGES.ps1` inside a `tracker\` folder.
Everything else is convenience or auto-generated.

## Get the files

### ✅ Recommended — Download ZIP (most reliable)

**[⬇ Download ZIP](https://github.com/redwan786/Drive-Activity-Tracker-Template/archive/refs/heads/master.zip)**

After unzipping, copy just the **`tracker`** folder into the drive/folder you want to track.
(Delete `README.md` / `TEMPLATE_README.md` from the unzipped copy — not needed.)

> The ZIP is the safest because the `#1- … .cmd` launcher names contain `#` and spaces,
> which break direct download URLs. The ZIP keeps them intact.

### Alternative — PowerShell (downloads scripts, builds the launchers locally)

Open PowerShell **inside the folder/drive you want to track** and paste:

```powershell
$b="https://raw.githubusercontent.com/redwan786/Drive-Activity-Tracker-Template/master/tracker"
New-Item -ItemType Directory -Force tracker | Out-Null
"INITIAL.ps1","UPDATE_CHANGES.ps1","RESET_CHANGES.ps1","SETUP.md" | % { Invoke-WebRequest "$b/$_" -OutFile "tracker\$_" }
@{"#1- INITIAL.cmd"="INITIAL.ps1";"#2- UPDATE_CHANGES.cmd"="UPDATE_CHANGES.ps1";"#3- RESET_CHANGES.cmd"="RESET_CHANGES.ps1"}.GetEnumerator() | % {
  "@echo off`r`ncd /d `"%~dp0`"`r`npowershell -NoProfile -ExecutionPolicy Bypass -Command `"Unblock-File -LiteralPath '%~dp0$($_.Value)' -EA SilentlyContinue; & '%~dp0$($_.Value)'`"`r`npause" |
  Set-Content "tracker\$($_.Key)" -Encoding ASCII
}
```

This downloads the 3 scripts + SETUP.md and **creates the `#1/#2/#3` launchers locally**
(avoiding the broken-URL problem). Then double-click **`tracker\#1- INITIAL.cmd`**.

## Case 1: First-time setup on a brand-new folder/drive

**Step 1** — Put the `tracker` folder (from above) inside the folder/drive you want to track.

**Step 2** — Open the `tracker` folder → **double-click `#1- INITIAL.cmd`**.

What it does:
- Creates the root `.gitignore` automatically
- Checks for `gh` (GitHub CLI)
  - **Not installed** → asks **"Install gh now? (Y/N)"**
    - **Y** → installs via winget (then close the window and run `#1- INITIAL.cmd` again)
    - **N** → prints a manual browser-based guide
- Asks the repo name — **Custom? (Y/N)** (N = auto from folder name)
- Creates a **Private** repo, pushes, and generates `README.md` + `CHANGES.md`

✅ Done.

### First-time GitHub login (only once per PC)

If you've never logged in to `gh` on this PC, a short flow appears. Answer:

```
? Where do you use GitHub?                         → GitHub.com
? What is your preferred protocol for Git ...?     → HTTPS
? Authenticate Git with your GitHub credentials?   → Yes
? How would you like to authenticate GitHub CLI?   → Login with a web browser
```

A one-time code shows in the terminal; the browser opens — confirm the code and
click **Authorize github**. You won't be asked again on this PC.

### About CHANGES.md on the first run

The first run only records a **baseline**, so `CHANGES.md` is empty at first — that's
normal. From the next change onward, every add/delete/rename/move is logged.

## Case 2: New Windows / re-install (the repo already exists)

The repo already holds the `tracker` folder, scripts, and git history — just clone it back.

**Step 1 — Clone into the exact folder/drive you track.**

Easiest with the terminal (replace the URL and target path):

```powershell
cd "C:\"                                  # or wherever the tracked root should be
git clone https://github.com/redwan786/<YOUR-REPO>.git .
```

- The trailing **`.`** clones **into the current folder** (no extra sub-folder).
- The target folder should be empty of git first. If it already has your data and
  you only need the tracker back, instead clone elsewhere and copy the
  **`.git`** folder + **`tracker`** folder + **`README.md`** + **`.gitignore`** into it.

**Using GitHub Desktop instead?**
`File → Clone repository` → choose the repo → set **Local path** to the folder you
track. If it makes a sub-folder, move everything (including the hidden **`.git`**)
up into the tracked root so `.git` sits beside your files.
(Turn on hidden files: **View → Show → Hidden items**.)

**Step 2** — Open `tracker` → **double-click `#2- UPDATE_CHANGES.cmd`**.

✅ Done — tracking continues exactly where it left off.

---

## Case 3: Scripts, `.gitignore`, or `.git` got deleted

- **Lost `.gitignore` or scripts** → just run **`#2- UPDATE_CHANGES.cmd`** (or `#1- INITIAL.cmd`);
  `.gitignore` self-recreates, and `git restore` brings back any tracked file.
- **Lost the whole `.git` folder / tracker** → re-clone the repo (Case 2).

## What `gh` is

`gh` is GitHub's official **command-line tool**. The setup uses it to create the
repo and log you in automatically. It's optional — say **No** at the prompt to get a
manual browser guide instead. One-time install · https://cli.github.com
